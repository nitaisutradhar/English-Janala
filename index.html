Frequently Asked Questions

1. The difference between `var`, `let`, and `const`
In JavaScript, var, let, and const are used for declaring variables, but they differ in scope and mutability: var is function-scoped and can be re-declared and updated, while let and const are block-scoped, with let allowing re-assignment and const preventing it.
2. The difference between `map()`, `forEach()`, and `filter()`
map() transforms each element of an array and returns a new array, forEach() executes a function for each element without returning anything, and filter() creates a new array containing only elements that meet a specific condition.
3. explain arrow functions and how they are different from regular functions
Arrow functions, introduced in ES6, offer a concise syntax for writing functions, particularly useful for short, simple functions, while regular functions are more flexible and have their own this binding, which is determined dynamically.
4. How JavaScript Promises work
A Promise in JavaScript is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. It is a modern alternative to callback functions, making asynchronous code more readable and manageable.
5. How closures work in JavaScript
A closure is a function that retains access to variables from its outer scope even after the outer function has finished executing. This happens because JavaScript uses lexical scoping, meaning functions remember the scope where they were created.
